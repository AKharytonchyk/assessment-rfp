## Engagement Strategy Matrix

Placement on the influence × interest grid drives the engagement strategy:

```
          LOW INTEREST         HIGH INTEREST
HIGH      ┌──────────────────┬──────────────────┐
INFL.     │  Keep satisfied  │  Manage closely  │
          │  (periodic exec  │  (weekly +       │
          │   brief)         │   decision mtg)  │
          ├──────────────────┼──────────────────┤
LOW       │  Monitor         │  Keep informed   │
INFL.     │  (quarterly      │  (sprint demo,   │
          │   update)        │   doc access)    │
          └──────────────────┴──────────────────┘
```

## Update rules

- Names and seniority populated within the first 3 business days of kickoff.
- Register reviewed weekly by DM; major changes (new stakeholder, exit, influence change) logged in the decision log.
- "Last engaged" column updated after every interaction on stakeholders in "Manage closely" or "Keep satisfied" buckets.
